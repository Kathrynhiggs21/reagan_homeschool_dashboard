GET /api/scheduled/drive-push/pending. For each row:

  a. Determine target Drive folder:

       - subfolder = row.subfolderName under that parent
       - If row.targetSubpath is non-empty (e.g. "2026-05"), nest one more level inside the subfolder. Create month folders on demand.
       - Final path: HubRoot / {canonicalParent} / {subfolderName} / {targetSubpath?} / {fileName}

  b. Resolve file content:
       - If row.contentText is non-empty AND row.mimeType starts with "text/", upload that text directly with the given mimeType. (No HTTP fetch needed — this is what the README seed rows and day_log markdown use.)
       - Otherwise download row.fileUrl. NOTE: this is a relative "/manus-storage/..." path which 302-redirects to an absolute presigned S3 URL — your HTTP client must follow redirects and include the Cookie header on the FIRST hop only (the redirect target is presigned and does not need the cookie).

  c. DEDUPE-ON-UPLOAD:
       - Before creating, search the target folder for a file with the same name.
       - If a same-named file exists, UPDATE that file's content (do not create a duplicate); reuse its driveFileId.

  d. POST { id, status: "pushed", driveFileId } back to /api/scheduled/drive-push/result.
     On failure: POST { id, status: "failed", errorMessage }.
     If row.targetFolder is one this playbook does not recognize, POST { id, status: "skipped", errorMessage: "unhandled target" } — never leave a row pending forever.
