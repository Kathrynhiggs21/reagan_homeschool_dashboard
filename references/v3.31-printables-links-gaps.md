# v3.31 — Printables reliability + agenda-link integrity (gap analysis)

Source of concern: Katy's worry that (a) printables can come out blank/broken/without an answer key, and (b) agenda links don't always open to the *exact* page, signed in and ready. Reinforced by a shared ChatGPT thread proposing a "Build → Audit → Fix" loop with capability checks and fallback ordering.

## What already exists (verified in code)

### Printables
- `agendaAssembler.ts` hydrates each block: curated lesson → daily printable → topic resources (`hydrateLessonForBlock`), and when all of those are empty, calls `synthesizeLessonForBlock` (LLM) to produce objectives + instructions + 3-6 practice items WITH answers + optional owned-book page ref. Result cached as an `assignmentsLibrary` row keyed on block id.
- `agendaPdf.ts` has a "never zero writable space" floor: a block with no URL/image/PDF/questions still gets 4 answer lines (worksheet branch) or a 10-line "Notes / Work Space" block (no-lesson branch). Answer key renders only when `L.answerKey` is non-empty.

### Links / sign-in
- `appLinkSignInMethodTagger.ts`: per-app canonical sign-in method (`google_sso` | `email_password` | `class_code`), preferred account role + email, kid badge, `canKidOpenNow`. Hard-blocks `reagan.higgs33@ihsd.us`; only `reaganhiggs910@gmail.com` for Reagan. Dad email intentionally not autofilled.
- `khanIxlDeeplink.ts`: deterministic Khan/IXL deep-link builder, subject + optional topic slug, `topicScoped` flag.

## The real gaps (what to fix)

### Printables
1. **Synthesis is all-or-nothing.** `synthesizeLessonForBlock` returns `null` on any LLM hiccup / schema reject / empty practice array. When null, the block prints as a bare title + blank Notes — pedagogically empty, no answer key. There is no retry and no deterministic offline fallback worksheet.
2. **No audit step.** Nothing verifies, after assembly, that every *content-bearing* block ended with real lesson content (curated, synthesized, book ref, or an explicit non-worksheet activity like an outing). Silent gaps are invisible to Mom until she opens the PDF.
3. **No standards anchor.** Synthesis prompt says "5th grade" but is not anchored to Ohio's Learning Standards, and no standard code is recorded on the synthesized worksheet.

### Links
4. **No reachability / freshness signal.** Deep links (esp. Khan topic-scoped slugs built from arbitrary topic text) can 404. Nothing validates that a built/curated link actually resolves, so a dead deep link can ship in the agenda.
5. **Sign-in readiness not surfaced in the agenda email/PDF.** The tagger exists but the nightly agenda doesn't clearly tell Mom which links are "tap-and-go for Reagan" vs "an adult must sign in first."

## Plan (v3.31)
- **Fix 1 — deterministic fallback worksheet:** add a pure, no-LLM `fallbackWorksheetForBlock(subject, durationMin)` that always yields 3-5 grade-5 practice items + answer key (number sense / reading-response / science-observation / writing-prompt by subject). `synthesizeLessonForBlock` retries the LLM once, then falls back to this so a content block is NEVER empty.
- **Fix 2 — assembly audit:** add `auditAgendaContent(input)` returning per-block status (`curated|synth|fallback|activity|EMPTY`) + a summary. Wire into the assembler; if any `EMPTY` content block remains, fire `notifyOwner` (best-effort) so Mom gets a heads-up the same night.
- **Fix 3 — Ohio standards:** thread an optional `standardCode` through synth + fallback (from the block's curriculum topic code where present) and print "Aligned to: 5.NBT.5" on the worksheet.
- **Fix 4 — link reachability tag:** pure `classifyAgendaLink()` that flags obviously-fragile links (deep slug with >N segments, known-404 patterns) and prefers the subject root when a topic slug is unverified; record a `linkConfidence` field. (No network calls in the pure helper; an optional best-effort HEAD check lives in the assembler.)
- **Fix 5 — sign-in readiness line:** surface the tagger's `canKidOpenNow` + badge next to each agenda link in the PDF/email ("Reagan: tap-and-go" vs "Grown-up signs in first").
- Tests for every new pure helper; full suite green; checkpoint.
